# Login Cracker — 登录爆破浏览器插件

> **⚠️ 免责声明：** 本工具仅用于 **授权安全测试** 和教学用途。未经授权对他人系统进行爆破属于违法行为，使用者需自行承担法律责任。

## 功能特性

- **自动识别**登录表单元素（用户名、密码、验证码输入框、验证码图片、登录按钮）
- **手动选取**页面元素（点击选取模式，支持手动编辑 CSS 选择器）
- **验证码处理**：自动刷新 + OCR 自动识别（Tesseract.js）或手动输入
- **模拟真实用户操作**：通过 DOM 事件模拟（focus → input → change → click），而非直接发送 HTTP 请求
- **每次尝试自动刷新验证码**
- **字典管理**：支持手动输入和加载本地 `.txt` 文件
- **数据持久化**：所有配置、字典、结果自动保存到`chrome.storage.local`，关闭后重新打开不丢失
- **攻击控制**：开始 / 暂停 / 继续 / 停止
- **实时日志与进度**展示

## 安装步骤

1. 打开 Chrome，访问 `chrome://extensions/`
2. 右上角开启 **开发者模式**
3. 点击 **加载已解压的扩展程序**
4. 选择本项目的 `login-cracker` 文件夹

## 使用指南

### 第一步：配置元素

1. 点击插件图标打开弹窗
2. 点击 **自动识别** 自动检测页面上的登录表单元素
3. 或使用 **选取** 按钮逐个手动指定：点击按钮 → 在页面上点击目标元素
4. 点击 **预览高亮** 查看选中效果

### 第二步：配置字典

1. 切换到 **字典** 标签页
2. 在文本框中手动输入用户名/密码（每行一个）
3. 或点击 **加载文件** 导入本地 `.txt` 字典文件

### 第三步：调整设置

1. 切换到 **设置** 标签页
2. 设置尝试间隔（建议 1500ms 以上避免被封）
3. 填写 **失败标识**（如"密码错误"）和 **成功标识**（如"欢迎"）
4. 根据目标页面勾选是否启用验证码

### 第四步：开始爆破

1. 切换到 **爆破** 标签页
2. 点击 **开始爆破**
3. 如果使用手动验证码模式，每次尝试会弹出验证码图片要求输入
4. 观察日志和进度
5. 随时可以 **暂停** 或 **停止**

## 工作原理

1. **Content Script** 注入到目标页面，负责所有 DOM 操作
2. **Popup** 作为控制面板，驱动攻击循环
3. 每次尝试：
   - 刷新验证码（点击验证码图片或修改 src 参数）
   - 捕获验证码图片 → OCR 识别或手动输入
   - 使用 `HTMLInputElement.prototype.value` 的原生 setter 填写表单（兼容 React / Vue / Angular）
   - 派发完整事件链：`focus → input → change → blur → mousedown → mouseup → click`
   - 等待响应并检测结果

## 注意事项

- 爆破期间请 **保持弹窗打开**（关闭弹窗会暂停攻击）
- 如果目标页面登录后会跳转，建议设置 **成功标识** 以便准确检测
- 间隔设置过短可能触发目标网站的防护机制
- 对于 HTTPS 页面的验证码图片，canvas 截取通常不会有跨域问题
- 部分使用 Shadow DOM 的页面可能无法自动识别元素，请手动填写选择器
